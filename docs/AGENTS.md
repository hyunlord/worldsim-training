# Documentation Rules

- Design documents belong in `docs/design/YYYY-MM-DD-*.md`.
- Execution and implementation plans belong in `docs/plans/YYYY-MM-DD-*.md`.
- Imported third-party or user-provided references belong in `docs/references/`.
- Keep documentation concise, dated, and aligned with the actual repo structure.
- When behavior changes, update the nearest design or plan doc that would mislead a new contributor.
